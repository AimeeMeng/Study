1、有抽象方法的一定是抽象类，但抽象类中不一定都是抽象方法。
2、子类 如果是非抽象类，则一定要实现父类中所有的抽象方法（override），但如果子类也是抽象类，就可以只实现一部分的抽象方法。
3、父类将接口中全部方法实现，而子类继承了这些方法，不用再实现一遍，当然也可以覆盖这个方法。若父类是抽象类，可能没有将接口的方法全部实现，这时子类如果不是抽象的，则必须将其他接口方法实现。
4、重写狮子类的方法覆盖父类的方法，要求方法名和参数相同，将方法体重新写一次，子类的返回值类型必须小于等于父类的返回值类型，子类的方法修饰符大于等于父类的。

重载是同一个类中的多个方法，有相同的方法名，但参数却不相同，返回值可不同。

5、“运行时多态”
virtual关键字用于指定属性或方法在派生类中重写.
默认情况下,派生类类从其基类继承属性和方法,如果继承的属性或方法需要在派生类中有不同的行为,则可以重写它,即可以在派生类中定义该属性或方法的新实现,这时在基类中该属性或方法必须用virtual关键字修饰.virtual关键字用于指定属性或方法时可以在派生类中重写.具有virtual关键字的方法或属性被称作虚拟成员.
class A
{
public virtual void F(){}
}

class B:A
{
public override void F(){}
}



但是参考网上资料，意识到还有一个问题：实现接口的类不会用到这个接口中所有的方法。。可以先写一个Adapter类实现这个接口，但实现方法内容为空；然后用区域类 建筑类去继承前面的adapter类，然后重写自己所需要实现的接口方法。
